安装gnome3

添加gnome overlay

  • emerge -av layman
  • echo "source /var/lib/layman/make.conf" >> /etc/make.conf
  • layman -L
  • layman -a gnome
  • mkdir /etc/portage
  • mkdir /etc/portage/package.keywords /etc/portage/package.unmask /etc/protage/profile
  • ln -s /var/lib/layman/gnome/status/portage-configs/package.keywords.gnome3 /etc/portage/package.keywords/
  • ln -s /var/lib/layman/gnome/status/portage-configs/package.unmask.gnome3 /etc/portage/package.unmask/
  • cat /var/lib/layman/gnome/status/portage-configs/use.mask.gnome3 >> /etc/portage/profile/use.mask
  • emerge -avuND world

安装xorg

  • 修改 /etc/make.conf ，添加下面两句：
    • INPUT_DEVICES="evdev synaptics"
    • VIDEO_CARDS="radeon"
  • echo "x11-base/xorg-server udev" >> /etc/portage/package.use
  • emerge -av xorg-server
  • emerge -av xterm
  • startx 成功

安装gnome3

  • emerge -av gnome
  • 根据反馈结果：
    • 不断修改 /etc/portage/package.use 。
    • 如果出现slot冲突，在 /etc/portage/ 中进行修改，然后emerge -avuND world。
  • emerge -av gnome 出现如下错误：
    • sys-fs/lvm2 报错，向 /etc/portage/package.use 添加sys-fs/lvm2 -static。
    • media-libs/gst-plugins-good-0.10.23 报错，这是因为23版本对gtk+3来说已经不适合了，所以，向 /etc/portage/package.accept-keywords 文件中添加media-libs/gst-plugins-good ~x86，这样会使用28版本代替23版本。
    • gnome-extra/zenity-3.0.0 报错，安装libnotify，emerge -av libnotify。

启动gnome3

  • /etc/init.d/dbus start
  • rc-update add dbus default
  • echo "exec gnome-session" > ~/.xinitrc
  • sed -i '1i\export XDG_MENU_PREFIX=gnome-' ~/.xinitrc
  • startx

思考

  • 我将gnome overlay加入了，然后，我将其中间的对单一portage的配置文件放入到了系统中，那么对系统造成的影响有：
    • 使用update的时候，会对现在有的软件进行一定的修改，如果当下有软件在单一portage文件中，那么必然需要升级，对于系统本来没有的软件，效果会反映在emerge -s上面。
    • 如果我没有立刻update，那么我现在安装的新的软件，会有一定的影响，比如依赖有新的版本，所以，先update。

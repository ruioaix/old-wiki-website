Package Management

  • 所有软件在fedora上都是RPM包。
  • 这里介绍使用yum和packagekit来管理rpm包。

yum

  • yum 是fedora的PM。
  • 它完成一般的PM的工作如RPM，同时，对于repo和package source有很强的灵活性。
    • 查询包的信息。
    • 从仓库下载包。
    • 自动处理依赖关系，安装卸载包。
    • 更新系统。
    • 配置新的仓库，包源。
    • 安装插件。
  • GPG貌似是一种额外的验证，使能了之后，yum不会安装没有这个标志的包。称这为安全PM。
  • yum比packagekit更全面和方便。
  • yum check-update
  • sudo yum update gdb
  • /var/log/yum.log是yum的日志文件。
  • sudo yum update
  • sudo yum search term,搜索包名字，描述，概要。
  • yum list glob_expression,就是使用*和？来代替，为了安全，在*和？之前加上反斜杠或者表达式加上双引号。
  • yum list all,安装的或者可用的。
  • yum list installed,在系统上安装的。
  • yum list available，所有使能的仓库中所有可用的包。
  • yum groulist,所有的包组。
  • yum repolist,所有的使能的仓库的信息。
  • yum info xxxx,可以使用表达式，显示一个包的一般信息。
  • yumdb info xxxx,显示包的硬件信息，如校验码等。
  • sudo yum history
  • sudo yum install xxx,安装包，以及他没有安装的依赖包。
  • yum provides "xx"
  • 包组有一个名字和一个ID，安装一个包组，就是安装一系列的软件来完成一个工作目的。
  • yum grouplist -v
  • yum groupinstall "完整的组名或者ID"
  • yum install @GROUPID
  • yum remove xxx,删除xxx，并且，任何依赖于xxx的包也都将被删除，yum不能够只删除包，而不删除依赖于他的包（rpm可以做到，但是这将是系统处于不稳定的状态）。
  • yum groupremove "完整的组名"
  • yum groupremove groupid
  • yum remove @groupid
  • yum 在删除包组的时候，会删除组里所有的包，但通过设置，可以不删除那些被别的包依赖的包。
  • yum的配置文件是/etc/yum.conf。
    • [main]是yum全局的控制。
    • [repository]和/etc/yum.repos.d/下面的.repo文件是对仓库单独的配置。
    • yum变量的控制。
    • 设置我自己的yum仓库。
  • [main]覆盖[repository]。
  • 推荐仓库的配置放在/etc/yum.repos.d/下，而不是写在[repository]下面。
  • yum的插件不建议都关闭，这是不明智的。
  • yum update --disableplugin=presto
  • yum install yum-plugin-security

packagekit

  • packagekit 使用yum的仓库作为他的包源。
  • 在packagekit中，是不可能添加或者删除yum的仓库的。

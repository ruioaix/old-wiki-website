Using UTF-8 with Gentoo

character encodings（字符编码）

  • 每一个字符在电脑里面都是使用一个number表示的，电脑本身是不认识字符的。
  • 最普遍的character set（字符集）是ASCII。
    • ASCII是7-bits的，0-127；包括33个不可见控制字符，0-31和127（del），32-126是可见字符，包括空格，标点，字母，数字。
    • 第八位原本是用来表示校验位，不需要的时候，设为0。
  • ISO 8859是对ASCII的一种扩展，其启用了第八位，得到了另外127（或者是128？）个字符，其用来表示欧洲带有重音的语系，现在有15个分支：ISO 8859-1到ISO 8859-15。
  • 非拉丁字母的字符集带来更多的冲突。
  • Unicode脱离了单子节字符集的限制，其使用17个位面，每个位面包含65536（16位）个代码点，一共可以描述1114112（17×65536）个字符。
    • 由于第一个位面包含了我们所用的几乎所有的字符，所以常常会错误的认为Unicode是双子节字符集。
    • Unicode有很多不同的映射方式，其中最普遍的是UTF和UCS。UTF-8是传播范围最广的方法。
  • UTF-8是长度可编的字符编码，表示一个字符，可以使用1-4bytes。
    • 第一个个byte用于ASCII，使其完全兼容。

setting up UTF-8 with Gentoo Linux

  • 确定locale -a有以.utf-8结尾的输出。
  • 如果没有：localedef -i zh_CN -f UTF-8 zh_CN.UTF-8
  • 编辑 /etc/env.d/02locale 文件，加入LANG="zh_CN.UTF-8"。
  • env-update
  • source /etc/profile

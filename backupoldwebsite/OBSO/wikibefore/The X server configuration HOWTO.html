The X server configuration HOWTO

  1. X window system是一份规范，在屏幕上勾画或者移动图形，键盘鼠标的使用等等的规范；其也被简称为X或者X11
  2. 使用X window system规范，是linux用户可以使用图形界面的应用是Xorg-X11，这是xfree86的一个分支
  3. 要安装xorg-x11,则需要配置好显卡
  4. 在gentoo中配置显卡，需要kernel modesetting的支持，这是新的特性，简称KMS，在内核中配置
  5. 对ATI显卡，如果使用开源的驱动，那么需要安装radeon-ucode
  6. 然后就是配置make.conf这个文件，在这个文件中决定使用那个显卡，输入设备等等。

安装X server

  • x server是安装gnome的基础，当然也是kde的。
  • 安装x server之前，需要对内核进行一些设置，从而支持输入设备和显卡。
    • 输入设备一般使用 evdev （通用的输入驱动），这个是在 Device Drivers -> _Input device support -> Event interface （使能）。
    • 现在的开源显卡驱动需要KMS（kernel modesetting）的支持。
      • 在 Device Drivers -> Graphics support -> Support for frame buffer devices 中，关闭所有的驱动，这是传统的frame buffer模式，和kms冲突。
      • 在 Device Drivers -> Generic Driver Options -> 选择 Include in-kernel firmware blobs in kernel binary ；设置 External firmware blobs 和 Firmware blobs root directory 。
    • 添加Radeon KMS support
      • 在 Device Drivers -> Graphics support -> Direct Rendering Manager 中添加 ATI Radeon 和 Enable modesetting on radeon by default 支持。
  • 由于我使用的是ATI的显卡，所以，我还必需安装emerge -av radeon-ucode。
  • 在配置好内核之后，我需要对 /etc/make.conf 文件中添加一些语句：
    • INPUT_DEVICES="evdev synaptics"
    • VIDEO_CARDS="radeon"
  • echo "x11-base/xorg-server udev" >> /etc/portage/package.use
  • emerge xorg-server
  • env-update
  • source /etc/profile

配置xorg

  • 一般来说，按照上面的步骤安装好了之后，应该就不需要什么设置了。
  • startx
    • 这是一个script，其启动x server和一些图形化应用。执行什么应用由以下逻辑决定。
      • 如果 .xinitrc 存在于用户主目录下，那么其就会执行其中的语句。
      • 如果没有上述文件，那么，startx就会读取 XSESSION 变量，然后根据变量执行 /etc/X11/Sessions/ 中的script。
      • 而 XSESSION 可以在 /etc/env.d/90xsession 中定义。
  • 执行startx之后，如果还没有窗口安装管理器（gnome或者kde），那么显示的就是黑屏，可以安装xterm什么的验证一下。
  • 配置xorg是非常不推荐的，应该作为迫不得已的一种做法，配置文件在 /etc/X11/xinit/ 中。

infrastructure services

services and daemons

  • systemctl 是用来开始，停止，重启服务；并设置开机启动的服务。
  • 貌似ntsysv和chkconfig也可以处理服务相关，但是，使用systemctl比较明智。
  • systemctl enable service_name.service，使能服务，下次系统启动的时候，服务就会启动。
  • systemctl status service_name.service，查看服务状态。
  • systemctl disable service_name.service，下次系统启动的时候，服务不再启动。
  • systemctl is-active service_name.service，status的缩水版。
  • systemctl list-units --type=service
  • systemctl start service_name.service，启动服务，但下次开机不一定。开机取决于enable。
  • systemctl stop service_name.service，停止服务。
  • systemctl restart service_name.service。

openssh

  • ssh是一种协议:两个系统间以client和server的形式进行安全通讯；允许用户远程登陆server。
  • ssh不同于ftp，telnet，其加密特性使得其非常安全。
  • ssh程序替代以前的那些远程登陆的软件，比如telnet和ssh。
  • scp程序替代以前的那些远程复制的软件，比如rcp。
  • 网络威胁主要有两种：
    • 两个系统之间的交流信息被截取了。
    • 信息被提交给了错误的目的地。
  • ssh加密了每个网络传输中的包，只有在local和remote系统才知道。
  • ssh使用数字签名验证目的地，不会找错地点。
  • ssh 目前有两个版本，1，2，2改进了1的缺点，最好使用2，但openssh支持1。
  • ssh的配置文件有两种，一种是client和server。
    • client是ssh，scp，sftp。
    • server是sshd。
    • 系统范围的配置文件在/etc/ssh/。
    • 特定用户的配置文件在~/.ssh/。
    • ~/.ssh/id_rsa~/.ssh/id_rsa.pub2版本的ssh的私人和公共key。
  • systemctl start sshd.service
  • systemctl stop sshd.service
  • 为了使用ssh，scp，sftp从client连接到server，必须生成验证key。
    • 生成RSAkey，2版本的：ssh-keygen -t rsa
    • 将产生的rsa.pub放到server的已验证key文件中，就可以登陆server中了。
  • ssh john@xxx.example.com
  • scp localfile username@hostname:remotefile，这是复制文件到server。
  • ssh username@hostname:remotefile localfile
  • sftp username@hostname
    • ls cd mkdir rmdir put(localfile to remotefile) get(remotefile to localfile)
